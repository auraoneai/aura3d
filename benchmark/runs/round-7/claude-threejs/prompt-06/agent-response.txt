## Summary

Built a **Mini-Golf Hole** app in `source/` using plain Three.js 0.165.0 (the provided context bundle is Three.js — there was no `llms.txt` or `benchmark-recipes.md`, and the bundle exposes no physics API, so the ball uses lightweight kinematic physics rather than a third-party engine).

**Implementation** (`source/src/main.ts`):
- **Green** — flat striped plane with a wooden perimeter border and a tee marker.
- **Obstacle** — a stone tower (cylinder + red cone roof) mid-green with circle-collision reflection.
- **Ball** — white sphere with velocity + rolling friction, wall/obstacle bounce, visual roll, and hole capture.
- **Aim & shoot** — click-and-drag a slingshot from the ball; a colored `ArrowHelper` shows direction and grows/heats with power; release to shoot.
- **Score counter** — HUD showing strokes + par, plus a "Holed out!" banner with click-to-replay.
- **Follow camera** — smoothly lerps to a fixed offset behind/above the ball and looks at it every frame.

## Commands

- **Build:** `npm install && npm run build` (runs `tsc --noEmit` typecheck + `vite build`)
- **Run:** `npm run dev` (Vite dev server) — or `npm run preview` to serve the production build

## Assumptions

- Context bundle is Three.js, not an Aura3D package — built directly on `three` public imports.
- No physics library is available in the bundle, so ball motion (friction, wall/obstacle bounce, hole sink) is implemented as simple custom kinematics.
- Par is fixed at 2; sinking the ball requires it to enter the hole below ~55% of max speed (faster shots lip out), which is standard mini-golf behavior.
- The hole is at the far end (−z), tee at the near end (+z), with the tower obstacle between them.

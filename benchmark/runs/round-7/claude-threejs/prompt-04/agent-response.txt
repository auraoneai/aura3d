## Summary

Built a **Neon Tunnel Flythrough** in `source/src/main.ts` using plain three.js (v0.165.0) from the context bundle.

**How each required visual is satisfied:**
- **Tunnel/tube interior** — `THREE.TubeGeometry` from a winding closed `CatmullRomCurve3` spline (600 segments), rendered with `side: BackSide` so the camera sees the interior wall.
- **Emissive segments** — a procedural canvas neon-grid texture used as `emissiveMap` on the tube wall (28×6 repeats), plus 90 emissive neon `TorusGeometry` rings (cyan/magenta/purple/green/pink/orange) oriented perpendicular to the path along the tunnel.
- **Bloom** — `EffectComposer` → `RenderPass` → `UnrealBloomPass` (strength 1.6, radius 0.6) → `OutputPass`, with ACES tone mapping.
- **Fog falloff** — `THREE.FogExp2` (density 0.045) so distant rings fade into dark depth.
- **Camera flythrough** — `setAnimationLoop` drives the camera along `curve.getPointAt(t)` looking slightly ahead, traversing the full loop every 36s, with a moving point light and drifting wall texture for motion.

## Commands

- **Install:** `npm install`
- **Build:** `npm run build` (runs `tsc --noEmit` then `vite build`)
- **Run:** `npm run dev -- --port <assigned-port>`

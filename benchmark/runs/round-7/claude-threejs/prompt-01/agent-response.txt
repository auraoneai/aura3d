## Summary

Built a **Physics Playground** in the clean `source/` directory using plain three.js 0.165.0 (the provided context bundle ships no physics engine, so a small self-contained rigid-body solver was written rather than pulling in an external one).

**What it does** — all required visual evidence is covered:
- **50 falling cubes** rendered as a single `InstancedMesh` (colored, shadow-casting), dropped in a loose grid above the ramp.
- **A tilted ramp** (20° `BoxGeometry` slab with a low-edge lip) that cubes land on and slide down.
- **Collision response** via semi-implicit Euler with substeps: cube↔ground and cube↔tilted-ramp oriented-plane contacts (restitution + friction + tumbling spin), and cube↔cube sphere/sphere impulse resolution.
- **Reset button** and a **live contact-count overlay** in an HTML HUD (updates every frame), plus an orbit hint.
- **Camera orbit** through `OrbitControls` (damping, clamped distance/polar angle) with a default framing that shows the whole setup at a glance.

**Files changed (inside `source/` only):**
- `src/main.ts` — the full app.
- `package.json` — added `@types/three@0.165.0` to devDependencies so the strict `tsc --noEmit` typecheck resolves three's types.

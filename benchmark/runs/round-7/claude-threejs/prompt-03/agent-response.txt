## Summary

Built a procedural solar system in `source/` using three.js v0.165 with addons from the context bundle (`OrbitControls`, `EffectComposer`/`RenderPass`/`UnrealBloomPass`, `CSS2DRenderer`).

**How each requirement is met:**
- **Sun + 6 planets** — one emissive sun sphere plus Mercury, Venus, Earth, Mars, Jupiter, Saturn (Saturn with a ring).
- **Different orbital distances/speeds** — radii 24→110 with faint orbit guide rings; inner planets orbit faster (each planet has its own angular speed on a pivot).
- **Readable labels** — `CSS2DRenderer` HTML overlay labels attached above the sun and every planet (crisp DOM text with styled background, always face the camera).
- **Bloom on the sun** — `UnrealBloomPass` with a high threshold (0.85) so only the bright sun (plus its additive glow halo) blooms; planets stay clean.
- **Orbit camera framing the system** — `OrbitControls` with damping, positioned at an elevated 3/4 angle (0, 70, 150) with min/max distance to keep the whole system in frame.

**Build command:** `npm run build` (runs `tsc --noEmit` + `vite build`)

**Run command:** `npm run dev -- --port <assigned-port>` (Vite dev server; `npm run preview` also serves the built `dist/`)

**Assumptions:**
- The context bundle is three.js (not Aura3D, despite the project CLAUDE.md); imported jsm addons via the standard `three/examples/jsm/...` paths, which resolve from the installed `three` package and match the files provided in `./context/examples/jsm`.
- "6 planets" interpreted as the classic inner set through Saturn; counts/labels are what matter, so distances/sizes are stylized (not to scale) for clear visual separation.
- Bloom is applied scene-wide but tuned (high threshold) so it visually affects only the sun.
- No assets used — everything is procedural geometry, per the prompt.

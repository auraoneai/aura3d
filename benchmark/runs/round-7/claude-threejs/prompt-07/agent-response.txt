## Summary

Built a procedural **Material Lab** with three.js v0.165.0 in `./source/src/main.ts`.

**What it renders:**
- **Five spheres**, evenly spaced in a row, each with a visually distinct material:
  - **Metal** — `MeshStandardMaterial`, `metalness: 1.0`, low roughness (sharp env reflections)
  - **Glass** — `MeshPhysicalMaterial` with `transmission: 1.0`, `ior: 1.5`, thickness (refractive/see-through)
  - **Rubber** — `MeshStandardMaterial`, `roughness: 0.95`, non-metal (matte diffuse)
  - **Emissive** — black base with bright orange `emissive` + `emissiveIntensity: 2.4` (self-lit, glows)
  - **Clearcoat** — `MeshPhysicalMaterial` with `clearcoat: 1.0` over a semi-rough red base (dual-layer glossy highlight)
- **Studio lighting:** 3-point setup (key `DirectionalLight` casting soft `PCFSoftShadowMap` shadows + radius, cool fill light, rim `SpotLight`) plus low ambient. A floor plane receives the soft shadows.
- **Environment map:** `RoomEnvironment` baked via `PMREMGenerator` into `scene.environment`, driving image-based reflections/lighting on all PBR materials.
- **OrbitControls** with damping enabled, target on the sphere row, min/max distance clamps.
- ACES Filmic tone mapping + sRGB output for correct studio look.

**Build command:** `npm install && npm run build` (run inside `./source`)

**Run command:** `npm run preview` (or `npm run dev`) inside `./source`

**Assumptions:**
- The context bundle is stock three.js (not the Aura3D engine referenced in CLAUDE.md), and contains no `llms.txt` or `benchmark-recipes.md`, so I built directly against three.js public APIs (`OrbitControls`, `RoomEnvironment`) resolved through the `three/addons/*` export path.
- No assets required — everything is procedural, satisfying the "built procedurally" constraint.
- Added a small CSS reset in `source/index.html` (inside the source directory) so the canvas fills the viewport.

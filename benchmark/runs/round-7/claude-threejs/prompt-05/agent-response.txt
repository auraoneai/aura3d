## How the required visual evidence is met

- **~36 bars** — `N=6`, a 6×6 loop creates exactly 36 box meshes.
- **Heights vary** — per-bar random target in `[0.5..6]` from a seeded deterministic RNG.
- **Animate from random values** — each bar grows from ~0 to its random target over 1.3 s (ease-out-cubic) in the render loop.
- **Color by height** — blue→green→red ramp via `Color.lerp`, set from each bar's own height.
- **Axis labels readable** — canvas-texture `Sprite` labels (no font assets): `X axis`, `Z axis`, `Height (value)`, numeric ticks 1–6 on X/Z and 0/2/4/6 on the height axis, plus a `GridHelper` and an HTML legend.
- **Hover-highlight** — `Raycaster` on `pointermove`: hovered bar is brightened + emissive-glowed and its coords/value shown in a live readout; restored on exit. Documented in `notes.md`.

## Assumptions

- This is the three.js arm; I used only the provided bundle (resolved `three`/`three/addons/*` through the bundle's `exports`) plus the official three.js public API — no CDNs or remote assets.

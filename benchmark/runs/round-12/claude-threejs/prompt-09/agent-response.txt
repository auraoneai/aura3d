## Summary

Built an animated primitive humanoid in `source/src/main.ts` using only `three` (0.165.0) from the context bundle.

**What it does:**
- **Humanoid from primitives:** sphere head, cylinder torso + neck, box hips, box arms (with sphere hands) and box legs (with box shoes) — assembled in a joint hierarchy so limbs pivot at shoulders and hips.
- **Procedural walk-cycle:** legs swing in opposite phase, arms swing counter to legs, knee-lift bend on the rising leg, vertical body bob per footfall, and a slight forward lean/side sway.
- **Moves across the ground plane:** the figure translates along +X (facing travel direction) and wraps around; a large green ground plane plus a grid make the forward motion obvious, with shadows cast onto the plane.
- **Pose implies animation:** the walk phase starts mid-stride (not a neutral T-pose) so the first rendered frame already shows a clear walking pose. Camera tracks the walking figure.

**Assumptions:**
- Raw Three.js benchmark (no Aura3D APIs); only `three` from the bundle is imported, no external assets.
- Vite TS scaffold left intact (`index.html`, `package.json` scripts, `tsconfig.json`); only `src/main.ts` was authored.
- Full-window canvas sized to `window.innerWidth/Height`.

**Commands for the runner:**
- Install: `npm install`
- Build: `npm run build`
- Run: `npm run dev -- --port <assigned-port>` (run from the `source/` directory)

## Summary

Built **Material Lab** as a raw Three.js (0.165.0) browser app in the clean source directory. The single entry point is `source/src/main.ts`.

**What it does (maps to required visual evidence):**
- **Five visible spheres** in a row, each with a distinct material:
  - **Metal** — `MeshStandardMaterial`, `metalness: 1`, low roughness, sharp env reflections
  - **Glass** — `MeshPhysicalMaterial` with `transmission: 1`, `ior: 1.5`, refractive/transparent
  - **Rubber** — `MeshStandardMaterial`, `roughness: 1`, non-metal, matte dark red
  - **Emissive** — `MeshStandardMaterial` with bright green `emissive` self-glow
  - **Clearcoat** — `MeshPhysicalMaterial` with `clearcoat: 1` glossy lacquer over a blue base
- **Studio lighting** — ambient + key directional (soft PCF shadows onto a ground plane) + cool fill + a back/rim spotlight, producing highlights and soft shadows.
- **Environment map** — procedural `RoomEnvironment` baked through `PMREMGenerator` into `scene.environment` (no external asset), giving reflections/IBL on every material.
- **Orbit controls** — `OrbitControls` with damping, distance and polar-angle limits.
- Renderer uses ACES filmic tone mapping, sRGB output, and soft shadow maps.

**Build command:** `npm run build` (runs `tsc --noEmit` typecheck + `vite build`)

**Run command (for the runner):** `npm run dev -- --port <assigned-port>`

## Summary

Built a procedural solar system in `source/src/main.ts` using only `three@0.165.0` and bundled official addons:

- **Sun + 6 planets** — emissive sun at center; Mercury, Venus, Earth, Mars, Jupiter, Saturn (with ring) as `MeshStandardMaterial` spheres lit by a central `PointLight`.
- **Different orbital distances & speeds** — distances 9 / 14 / 20 / 27 / 38 / 50 with faint orbit rings, each on its own pivot group rotating at a distinct angular speed.
- **Readable labels** — `CSS2DRenderer` + `CSS2DObject` HTML chips attached to the sun and every planet, rendered in a crisp overlay unaffected by bloom.
- **Bloom on the sun** — `EffectComposer` → `RenderPass` → `UnrealBloomPass` with a luminance threshold (0.65) so only the bright sun + additive halo bloom; planets stay sharp.
- **Orbit camera** — `OrbitControls` with damping and gentle auto-rotate; camera pulled back/angled (`0, 42, 78`) to frame the entire system (outermost orbit at radius 50).
- Starfield backdrop and full-viewport canvas styling in `index.html`.

## Commands

- **Build command:** `npm run build` (runs `tsc --noEmit` typecheck + `vite build`)
- **Run command (for the runner):** `npm run dev -- --port <assigned-port>`

Both run from the `source/` directory. `npm install` was already executed.

## Assumptions

- This is the raw **Three.js** benchmark side (per `llms.txt`), so I imported from `three` and `three/addons/*` (matching the v0.165.0 bundle in `context/`); no Aura3D APIs were used.
- No `benchmark-recipes.md` existed in the context bundle, so the app was built directly from the documented addons.
- Planets are procedural spheres with representative (not physically accurate) sizes/distances/colors, compressed for clear on-screen framing; start angles are randomized so planets aren't collinear at load.
- Bloom is global but threshold-keyed to the bright sun, satisfying "bloom on the sun" while keeping planet surfaces and labels readable.

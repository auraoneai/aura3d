## Summary

Built a raw Three.js 3D data visualization in `source/` (Vite + TypeScript strict, `three@0.165.0`).

**What it does:**
- **36 bars** — a 6×6 grid of `BoxGeometry` bars, geometry translated so each grows upward from the floor (`scale.y`).
- **Animated heights** — each bar eases toward a random target (framerate-independent), and all targets re-randomize every 3.5s, so heights continuously animate and vary.
- **Color by height** — an HSL ramp (blue → cyan → green → yellow → red) recomputed each frame from each bar's normalized height.
- **Hover-highlight** — `Raycaster` on `pointermove`; the hovered bar gets a white emissive glow + a slight X/Z scale-up, the cursor switches to a pointer, and the HUD readout shows its height. Highlight clears on pointer-out.
- **Orbit camera** — `OrbitControls` with damping, polar/zoom limits.
- **Axis labels** — readable HTML overlays via `CSS2DRenderer`: X1–X6 (front edge), Z1–Z6 (left edge), Y value ticks 0–5 with a "VALUE" title, plus "X AXIS" / "Z AXIS" titles.

I added `@types/three@0.165.0` to `source/package.json` devDependencies (needed because the build runs `tsc --noEmit` and `three` ships no bundled types). Only files inside `source/` were modified.

**Build command:** `npm run build` (runs `tsc --noEmit` then `vite build`)

**Run command (for the runner):** `npm run dev -- --port <assigned-port>`

**Assumptions:**
- The benchmark bundle is raw Three.js (per `context/llms.txt`); no `benchmark-recipes.md` exists, so no recipe was copied.
- Used the bundled official addons `three/addons/controls/OrbitControls.js` and `three/addons/renderers/CSS2DRenderer.js`.
- Bars range ~0.4–5.0 in height; re-randomizing every 3.5s keeps the animation continuously visible. Hover-highlight uses emissive glow + a subtle scale bump and a live height readout in the HUD.

## Summary

I built a procedural **Mini-Golf Hole** in raw Three.js (v0.165.0) in the source directory (`src/main.ts`). No external assets — everything is generated from Three.js primitives.

**What it implements (all required visual evidence):**
- **Flat green** — a striped box-geometry turf, bounded by wooden walls.
- **One obstacle** — a red brick block in the middle of the fairway, with real circle-vs-AABB collision (ball bounces off it).
- **Ball with physics** — a white sphere with velocity integration, frictional damping, rolling rotation, wall reflections, obstacle bounces, and a hole/cup with a flagpole. Sinking the ball shows a "Holed in N strokes" message and resets on next click.
- **Click-to-aim-and-shoot** — pointer-drag slingshot from the ball; a colored aim line + arrowhead (yellow→red by power) shows direction and strength; release launches the ball. Strokes increment per shot.
- **Score counter** — an HTML HUD overlay (top-left) showing "Strokes: N".
- **Follow camera** — the perspective camera smoothly lerps to a behind/above offset and keeps the ball framed every frame.

**Note on scaffold change:** `three@0.165.0` ships no TypeScript declarations, and the fixed `build` runs `tsc --noEmit` first, which would fail on the untyped import. I added `@types/three@0.165.0` to `devDependencies` (within the source dir only) so the unchanged build command passes.

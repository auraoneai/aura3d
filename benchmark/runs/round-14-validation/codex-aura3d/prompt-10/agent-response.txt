Implemented the Aura3D product viewer in `source` using the typed `assets.sneaker` asset and the matching product-viewer prefab recipe. Build completed successfully.

Build command:
```bash
cd source && npm run build
```

Run command for the runner:
```bash
cd source && npm run dev
```

Assumptions:
- The provided asset is `source/public/benchmark/assets/sneaker.glb`, corresponding to the allowed `benchmark/assets/sneaker.glb`.
- `npx @aura3d/cli@latest` was unavailable from npm, so I used the local Aura3D engine package’s asset-registration implementation to generate `src/aura-assets.ts` and hashed public asset output.

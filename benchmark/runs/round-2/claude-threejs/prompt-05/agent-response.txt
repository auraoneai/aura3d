## What was built

A **6×6 grid of 36 bars** rendered with Three.js 0.165.0 (`source/src/main.ts`, UI/styles in `source/index.html`):

- **36 bars** — a 6×6 grid of box meshes, base-anchored so they scale upward.
- **Animated heights from random values** — each bar eases toward a random target height every frame; bars grow in from flat on load, re-randomize via the **Randomize values** button, and auto-shuffle every 4.5s.
- **Color by height** — a blue → green → red gradient (`colorForHeight`) recomputed each frame from the bar's current height.
- **Hover-highlight** — raycasting against the bars; the hovered bar brightens, gains an emissive glow, and thickens slightly, with a bottom-center readout showing its cell ID (e.g. `C4`) and live value.
- **Orbit camera** — `OrbitControls` with damping (drag to orbit, scroll to zoom).
- **Axis labels** — crisp HTML labels via `CSS2DRenderer`: column letters (A–F) on X, row numbers (1–6) on Z, a vertical value (Y) ruler with numeric ticks, plus axis titles.

## Commands

All run from `source/`:

- **Install:** `npm install`
- **Build:** `npm run build` (runs `tsc --noEmit` typecheck, then `vite build`)
- **Run (dev):** `npm run dev` — or preview the production build with `npm run preview`

## Run notes — hover-highlight

Move the pointer over any bar. On each frame the app casts a ray from the cursor through the camera into the bar meshes; the nearest hit becomes the "hovered" bar. That bar is **lightened, given an emissive glow, and scaled up ~12% in X/Z**, while the previously hovered bar is restored to its height-based color and size. A floating readout at bottom-center displays the bar's grid cell (column letter + row number) and its current value, updating live as the bar animates. Moving off all bars (or off the canvas) clears the highlight and hides the readout.

## Assumptions

- **Built procedurally**, no assets, as required.
- I added `@types/three@0.165.0` (exact match to the pinned `three@0.165.0`) to `devDependencies`, because the `three` package ships no `.d.ts` files and the scaffold's `build` script runs `tsc --noEmit` under `strict`. Without it, typecheck/build would fail. This is the only dependency change; it's a standard TS types package, not a fetched asset.
- Only files inside `source/` were modified (`src/main.ts`, `index.html`, `package.json`, plus generated `node_modules`/`dist`/`package-lock.json`).
- Color maps to a fixed value scale (0–6) so color stays consistent across re-randomizations rather than relative to the current min/max.

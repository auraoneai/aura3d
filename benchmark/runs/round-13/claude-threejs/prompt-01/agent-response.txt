## Summary

Built a **Physics Playground** in raw Three.js (0.165) at `source/`, satisfying all required visual evidence.

**What it does**
- **50 procedural cubes** spawn stacked above the high end of the ramp and fall under gravity.
- **Tilted ramp** (24° rotated slab) with collision response — cubes cascade down its face and pile at the base.
- **Custom fixed-timestep solver** (`1/120 s` substeps): cube↔cube contacts (sphere approximation with positional correction + impulse exchange), and cube↔ramp/floor/wall resolution against oriented half-spaces with restitution and friction. Three.js bundles no physics engine and none was in the context, so a lightweight solver was implemented; invisible bounding walls keep the pile in frame.
- **Camera orbit** via `OrbitControls` (auto-rotating, damped) framing the whole setup.
- **HTML overlay** with a **live contact count** (updated every frame) and a **Reset Drop button** that re-drops all cubes.
- Shadows + `RoomEnvironment` lighting for depth/readability.

**Files changed (inside `source/` only)**
- `src/main.ts` — the full app.
- `package.json` — added `@types/three@0.165` to devDependencies (required because `three` ships no TS types and the scaffold builds with `tsc --noEmit` in strict mode).

**Commands**
- Install: `npm install`
- Build: `npm run build`  *(runs `tsc --noEmit` then `vite build`; verified passing)*
- Run (for the runner): `npm run dev -- --port <assigned-port>`

**Assumptions**
- Added `@types/three` (matching three 0.165.0) so the strict-TS scaffold typechecks; this is a dev-only type dependency, not a runtime/asset addition.
- Cubes are treated as bounding spheres for cube-cube contacts — a deliberate simplification appropriate for a from-scratch solver; it produces convincing cascading and piling rather than exact box-face stacking.
- Invisible bounding walls (±9 on X/Z) keep cubes within the camera frame so the pile stays understandable in a screenshot.
- Camera auto-rotate is on by default to demonstrate orbit; user drag/scroll still works and overrides it.

## What was built

A procedurally-constructed humanoid placeholder walking across a ground plane, using only `@aura3d`-equivalent Three.js public imports (`three` + `three/addons/controls/OrbitControls.js`):

- **Head** — `SphereGeometry` (plus a partial-sphere cap so it reads as facing forward) on a short cylinder neck.
- **Torso** — tapered `CylinderGeometry`.
- **Arms & legs** — two-segment limbs built from `BoxGeometry` (upper + lower), each hinged at a shoulder/hip pivot group and an elbow/knee pivot group; legs have box feet.
- **Ground** — a 60×60 `PlaneGeometry` with a subtle grid and shadow receiving.
- **Walk cycle** — fully procedural in `updateWalk()`: hips and shoulders swing in opposition (`sin(phase)` vs `sin(phase + π)`), knees flex on the rear/lift portion, elbows hold a relaxed bend, the torso bobs vertically and counter-rotates, and the root advances along Z, turning around (180°) at the extents. It starts at `INITIAL_PHASE = 1.1` so the very first frame is already mid-stride.
- Soft-shadow directional sun + hemisphere fill, ACES tone mapping, sky background/fog, and orbit controls.

Verified: figure reads clearly as humanoid, all four body regions are primitives, ground plane visible with cast shadow, and the captured pose shows an obvious stride (not a static T-pose).

## Commands

Run these from the `source/` directory:

- **Install:** `npm install`
- **Build:** `npm run build`  (runs `tsc --noEmit` typecheck, then `vite build` — both pass)
- **Run (dev):** `npm run dev -- --port <assigned-port>`  (Vite dev server on `127.0.0.1`)
- **Preview built output:** `npm run preview`

## Assumptions

- Used the scaffold's pinned `three@0.165.0` from npm and imported OrbitControls via the package's `three/addons/...` path (resolves to the same addon file shipped in the context bundle); no external asset fetching.
- "Across the ground plane" is implemented as a continuous back-and-forth stroll with a turn at each end, keeping the figure on-camera while still translating across the plane.
- Camera/scene tuned so a single screenshot satisfies all evidence requirements; OrbitControls let a reviewer orbit the walking figure interactively.
- Headless render was done with software GL (SwiftShader) since this machine's headless Chrome has no GPU path; a real browser uses hardware WebGL with the same code.

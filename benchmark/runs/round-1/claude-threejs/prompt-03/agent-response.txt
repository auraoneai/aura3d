## Summary

I built a procedural solar system in `source/src/main.ts` using only the provided Three.js context bundle (three 0.165.0). I added `@types/three@0.165.0` to `package.json` devDependencies because the scaffold's `build` runs a strict `tsc --noEmit` typecheck that needs type definitions.

**How each requirement is met:**
- **1 sun + 6 planets** — a bright unlit sun sphere at the origin plus 6 textured `MeshStandardMaterial` planets (Ferro, Verdia, Azura, Rubex, Titanus, Cryos).
- **Visibly different orbital distances/speeds** — orbital radii 9/14/20/26/32/38 units, each with a faint visible orbit ring; inner planets orbit faster (0.62 → 0.15 rad/s, loosely Keplerian).
- **Readable labels** — `CSS2DRenderer` HTML labels overlaid on every planet and the sun, styled with high-contrast backgrounds so they stay legible.
- **Sun bloom/glow** — `EffectComposer` → `RenderPass` → `UnrealBloomPass` (threshold 0.85 so only the bright sun blooms), plus an additive corona sprite.
- **Orbit camera framing the system** — `OrbitControls` (damped) with the camera at (0, 48, 92) looking at the origin, framing all orbits out to radius 38.

Extras: procedural canvas-generated planet textures, a Saturn-like ring on the gas giant, a starfield background, and full resize handling.

**Commands** (run from `source/`):
- Install: `npm install`
- Build: `npm run build`
- Run (dev): `npm run dev -- --port <assigned-port>` (serves on `127.0.0.1`)

**Assumptions:**
- Added `@types/three@0.165.0` (dev-only) so the scaffold's strict typecheck passes; it pins to the same three version already in the scaffold.
- Planet names, colors, sizes, and tilts are invented (the prompt requires a *procedural* system, not the real one).
- Addons are imported via `three/examples/jsm/...`, which is what the provided context bundle ships and what the three package's exports map resolves.

## Mini-Golf Hole — done

The full app is in `source/src/main.ts` (single self-contained Three.js module, no addons). I left the scaffold's `package.json`, `index.html`, and `tsconfig.json` as-is.

### Commands (run from `source/`)
- **Install:** `npm install`
- **Build:** `npm run build` (runs `tsc --noEmit` typecheck + `vite build`)
- **Run:** `npm run dev -- --port <assigned-port>` → open the printed `http://127.0.0.1:<port>/`

### How it maps to the required visual evidence
- **Green** — a striped flat green bordered by raised wooden bumper walls, on a wider grass apron.
- **One obstacle** — a blue block mid-fairway that the player must steer around; the ball physically bounces off it.
- **Ball** — a white sphere with cast shadows that rolls (with visible spin) under physics.
- **Aim/shot interaction** — click-and-drag from the ball: a colored aim arrow points toward the cursor and a bottom power bar fills with pull distance; release to shoot. Each shot increments the stroke count.
- **Score counter** — HUD panel showing `Strokes: N` (Par 3), plus a "Holed in N!" banner with under/over-par result on success.
- **Camera** — smoothly follows the ball from behind-and-above, looking down the fairway toward the cup.

The cup has a white rim and a waving flag; sinking the ball (reaching the hole below the sink-speed threshold, with a slight lip-in assist) triggers the win banner, and a click resets to the tee.

### Physics model
Custom 2D rolling physics in the XZ plane: velocity integration with constant rolling friction, circle-vs-AABB collision resolution (penetration push-out + restitution bounce) against all bumpers and the obstacle, run on a fixed 120 Hz timestep accumulator for stability.

### Assumptions
- Built procedurally with no external assets, as the prompt requires.
- The context bundle is the Three.js library itself; `three@0.165.0` resolves from npm per the scaffold's `package.json`.
- "Click-to-aim-and-shoot" is implemented as a drag-to-aim/release-to-shoot sling with a power meter (a click with no drag does nothing, so accidental clicks don't waste strokes).
- Par is fixed at 3 for the single hole; the win banner reports the score relative to it.

## Summary

I built a procedural solar system in `source/` using **only `@aura3d/engine` public imports** (`scene`, `primitives`, `lights`, `material`, `effects`, `camera`, `interactions`, `createAuraApp`) — no assets, fully procedural.

**What's in the scene** (`source/src/main.ts`):
- **Sun** — emissive sphere at the origin, added first so it's the primary `effects.bloom` anchor → a strong warm glow.
- **6 planets** (Mercury → Saturn) as PBR spheres at clearly different orbital radii (3.0 → 10.0) and distinct sizes/colors, lit by a central point light at the sun (giving realistic phase shading).
- **Orbit rings** — each orbit drawn as a ring of small dim dots along a tilted orbital path, making the differing distances unmistakable.
- **Labels** — each planet + the sun gets a readable HTML overlay label (name + orbit radius + relative orbital speed, e.g. `orbit r=5.9 · 0.71×`). The engine has no 3D text primitive, so labels are anchored by reproducing the orbit camera's exact projection math in screen space.
- **Orbit camera** (`camera.orbit`) + `interactions.orbit`, framing the whole system with margin. A faint starfield sits behind it.

I verified the render in headless Chrome: `backend=webgl2`, no errors, sun bloom visible, all 6 planets + labels framed correctly.

### Commands
- **Install:** `npm install`
- **Build:** `npm run build`  (runs `tsc --noEmit` typecheck + `vite build`)
- **Run (dev):** `npm run dev -- --port <port>`  (e.g. `npm run dev -- --port 5193`)
- **Preview production build:** `npm run preview`

### Assumptions / notes
- **Labels via HTML overlay.** The `@aura3d/engine` public API has no text/label/sprite node, so labels are a DOM layer over the canvas, positioned by replicating the engine's deterministic orbit-camera projection (verified pixel-aligned to the rendered bodies). The 3D scene itself is built entirely with engine imports.
- **Static orbital positions.** The engine renders a declarative scene snapshot and only animates the camera (dolly) — there is no public per-object animation hook, so planets are placed at fixed angular positions rather than live-orbiting. "Different speeds" is therefore expressed via Kepler-style relative-speed values (∝ 1/√r) shown in each label rather than motion. Radii/sizes were tuned so the orbit camera frames all bodies with margin and the sun stays the dominant body.
- **`three` is bundled** as a dependency of the engine tgz; the engine's Three.js renderer path provides the bloom. Build emits a >500 kB chunk warning (Three.js) — harmless.
- No files outside `source/` were modified.

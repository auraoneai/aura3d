## Summary

Built a procedural **particle fountain** in raw Three.js (per `context/llms.txt`), entirely in the clean `source/` directory.

**What it does**
- **Emitter** — a metallic nozzle cone + pulsing glow marks the fountain origin.
- **Particles** — a `THREE.Points` system (6000-particle pool) launched upward in a slight cone, integrated under gravity (`-9.8 u/s²`) each frame, so they form a real **upward-and-falling arc**.
- **Ground collision** — particles bounce off the ground plane at `y=0` with restitution + friction, then settle/retire.
- **Lifetime color** — per-vertex color buffer ramps young→old (warm yellow → cyan → blue); an on-screen gradient legend labels it.
- **Emission-rate control** — a visible HTML range slider (0–3000 particles/s) drives a time-accumulated emitter.
- Slow auto-orbit camera, ground plane + grid, resize handling.

**One change to the scaffold:** the build runs strict `tsc --noEmit`, and `three@0.165` ships no type declarations, so I added `@types/three@0.165.0` as a devDependency in `source/package.json`. No files outside `source/` were touched.

### Build command
```
npm install && npm run build
```
(run from the `source/` directory)

### Run command (for the runner)
```
npm run dev -- --port <assigned-port>
```

### Assumptions
- This is the **raw Three.js** track (per `context/llms.txt`), not Aura3D — built with `three` core only, no addons needed.
- `@types/three@0.165.0` added to satisfy the strict typecheck step; it matches the pinned `three@0.165.0`.
- Emission rate defaults to 1200 particles/s; slider range 0–3000. Fixed 3 s particle lifetime, gravity, and a bouncing ground at `y=0`.
- Simulation runs on the CPU with a clamped delta-time (≤33 ms) for stability across frame-rate dips.

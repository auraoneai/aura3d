# PRD Amendment Template

Use this template for any standard change after Phase A is signed off.

The git commit message must start with:

```text
PRD-AMENDMENT: <short description>
```

## Required Body

Reason:

Files changed:

Prior result invalidated:

New benchmark round required: yes

User approval: `gchahal1982`, `<date>`

## Rule

An amendment that does not use the `PRD-AMENDMENT:` prefix, does not identify the invalidated result, or does not include user approval does not change the release standard.

## Standard Changes

- Particle prompts now require a real emission-rate input/control. Label-only
  buttons are explicitly invalid.
- `prefabs.particleFountain(...)` exposes emission-rate, gravity,
  ground-collision, and lifetime-color evidence.
- Neon tunnel defaults and recipes now target an inside-the-tube first frame
  with receding rings, wall chords, fog, and bloom.
- Data visualization recipes now require DOM title, axis, tick, and readout
  text in addition to the 3D bar geometry. The data prefab now uses grounded
  legend swatches and selected hover geometry instead of top-level trend
  strands that can read as stray scene artifacts.
- City day/night recipes now require the 3D scene to change sky/background,
  lighting, windows, street lights, and state markers. Text-only toggles are
  explicitly invalid.
- Material swatches have stronger emissive intensity and glow evidence.
- Humanoid motion-trail defaults no longer create detached previous-head or
  previous-torso ghost geometry, and the `walk` clip now uses whole-body motion
  so primitive limb segments stay connected in screenshot frames.
- Product prompts can use `prefabs.productViewer(assets.product)` to keep typed
  asset usage, clean staging, plinth, contact shadow, softboxes, and turntable
  evidence together.
- Prompt 10 asset metrics now allow hashed `/aura-assets/sneaker.<hash>.glb`
  outputs generated by the Aura CLI from the provided
  `benchmark/assets/sneaker.glb` source asset only when typed evidence is
  present: generated `src/aura-assets.ts`, app import from `./aura-assets`,
  `assets.sneaker` usage, matching `aura.assets.json` source/output hashes,
  and no `unsafeModelUrl(...)`. The runner-copied
  `public/benchmark/assets/sneaker.glb` fixture remains allowed only when its
  hash matches the prompt asset. Hand-written GLB URLs and unrelated copied
  assets remain invalid.

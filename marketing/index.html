01 — Workflows

Workflows,
not primitives.

Product viewer. Material studio. Asset inspector. Animation lab. Scene showcase. Each one a first‑party API, not a folder of demo code you have to keep in sync with the renderer underneath.

Pick a preset. Load a model. Ship.

const viewer = await createProductViewer({
  canvas, asset, environment,

  postprocess: { toneMapping: "aces", bloom: true }
});

05 — Migration

Move from Three.js.
Without moving everything.

A real migration is a long migration. @aura3d/three-compat lets your existing Three.js code keep running while you move the parts that matter — at your pace, on your schedule.

Same-scene comparison routes prove the parity, file by file.

// Existing Three.js imports keep working.
import { Scene, PerspectiveCamera, Mesh } from "@aura3d/three-compat";

// Reach for the workflow APIs when you're ready.
import { createProductViewer } from "@aura3d/engine/production-runtime";

Under the hood

Twenty-three packages.
One engine.

Renderer, scene, assets, animation, physics, controls, input, audio, materials, environments — every layer is a first‑party package, with its own tests, its own API surface, and its own report. Import what you need. Tree-shake the rest.

@aura3d/engine/rendering

WebGL2 + WebGPU devices, forward pass, queues, shadows, postprocess, instancing, diagnostics.

@aura3d/engine/scene

Object3D-style hierarchy, transforms, cameras, lights, renderables, serialization.

@aura3d/engine/assets

glTF / GLB, Draco, Meshopt, KTX2 / Basis, HDR / EXR, OBJ, render-resource conversion.

@aura3d/engine/animation

Clips, tracks, mixers, layers, skinning, morphs, IK, root motion, motion-quality reports.

@aura3d/engine/physics

Rigid bodies, colliders, raycasts, constraints, character controllers, scene sync, debug.

@aura3d/engine/math

Vectors, matrices, quaternions, rays, bounds, planes, frustums, projection — branded types.

@aura3d/engine/materials

PBR, transmission, clearcoat, sheen, specular, volume. Material extensions, end to end.

@aura3d/engine/environments

HDRI environments, PMREM, image-based lighting, environment registry and presets.

@aura3d/engine/controls

Orbit, trackball, fly, first-person, drag, transform, map, pointer-lock, picking.

@aura3d/engine/input

Keyboard, pointer, gamepad, gestures, virtual joystick, WebXR sessions, AR hit-test.

@aura3d/engine/audio

Spatial audio, mixer, effects, scene bridge, adaptive music, reverb and filters.

@aura3d/engine/workflows

Product configurator, material studio, asset viewer, scene showcase — composable.

@aura3d/engine/product-studio

Product viewer pipeline — asset import, presets, screenshots, diagnostics, variants.

@aura3d/engine/editor-runtime

Commands, history, gizmos, selection, prefabs, timeline, project serializer.

@aura3d/engine/ecs

Entity / component / system, archetypes, queries, command buffer, system scheduler.

@aura3d/engine/debug

Draw-call tracker, GPU profiler, render-state inspector, helpers, report exporter.

@aura3d/engine/scripting

Lifecycle scripts, scheduler bindings, sandboxed eval, scripting host.

@aura3d/engine/three-compat

Three.js-shaped imports backed by Aura3D. Move at the pace your team can absorb.

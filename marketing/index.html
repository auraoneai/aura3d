01 — The SDK

Workflows, not primitives.

Product viewer. Material studio. Asset inspector. Animation lab. Scene showcase. Each is a first-party API — not a folder of glue you keep in sync with the renderer underneath. Pick a preset. Load a model. Ship.

product-viewer.ts material-studio.ts animation-lab.ts
TypeScript 
// One workflow. Production defaults baked in.
import { createProductViewer } from "@aura3d/engine/production-runtime";

const viewer = await createProductViewer({
  canvas,
  asset:       "/models/concept-car.glb",
  environment: "studio-product",
  camera:      { preset: "product-hero", orbit: true },
  lighting:    { ibl: true, shadows: true },
  postprocess: { toneMapping: "aces", bloom: true },
  diagnostics: "report",
});

viewer.render();
const report = await viewer.capture({ screenshot: true });
// → { drawCalls: 47, fps: 60, triangles: 184k, mem: 38mb }

05 — Backends

Honest about what runs.

WebGL2 is the broad default. WebGPU is available where the browser grants an adapter — and surfaced explicitly when it doesn't. No silent fallback. No vague support badge. The diagnostics name the reason.

● Default · broad support

WebGL2

Battle-tested forward renderer with state caching, render queues, opaque/transparent sorting, culling, instanced rendering, render targets, postprocess, shadows, and full disposal paths.

06 — Migration

Bring your code. Move at your pace.

@aura3d/three-compat lets your existing imports keep running. Swap one line, your scene still renders. Graduate to workflow APIs when you're ready — file by file, route by route.

Today · what you have

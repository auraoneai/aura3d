# Claims And Boundaries

Allowed claims:

- Aura3D is a library and asset deployment pipeline for agent-written browser 3D.
- Agents write normal TypeScript/JavaScript source code.
- Users bring assets; Aura3D validates, hashes, indexes, type-generates, and
  checks deployment.
- Starter templates cover product viewer, cinematic scene, and mini game.
- Devtools expose asset state, FPS, draw calls, render size, backend, warnings,
  and errors.

full engine-level control when low-level renderer code or a framework-specific layer is a
better fit.

For cinematic scenes, use `docs/agents/cinematic-scene-quality.md`. The allowed claim is realtime previs-style browser scenes written as editable TypeScript with public Aura3D APIs, typed assets, camera motion, lighting, materials, atmosphere, diagnostics, and deployment checks.

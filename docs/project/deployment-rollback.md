# Deployment Rollback Plan

Version: 1.0.0

## Scope

This plan covers the local static demo artifacts generated by `pnpm build:external-demos` under `release-artifacts/external-demos/1.0.0/`.

It is not a public hosting, CDN, or production operations claim. V4 public deployment remains blocked until this static export is deployed to a durable public HTTPS origin and `G3D_PUBLIC_DEMO_URL=https://... pnpm verify:public-demo-deployment` validates that the public bytes match the static integrity manifest. The older per-demo URL manifest check in `docs/examples/external-demo-urls.json` / `pnpm verify:external-demos` can still provide additional public URL evidence, but it is not the V4 production-readiness gate.

## Integrity Evidence

`pnpm build:external-demos` writes `static-integrity-manifest.json` next to the exported demos. The manifest records:

- package version;
- source git SHA;
- exported HTML and JavaScript files;
- SHA-256 for every exported file;
- this rollback plan path.

## Rollback Procedure

1. Keep the previous `release-artifacts/external-demos/<version>/` directory immutable after validation.
2. Before promoting a new static export, compare `static-integrity-manifest.json` with the files staged for deployment.
3. If validation fails, restore the previous version directory as the active static root.
4. Re-run `G3D_PUBLIC_DEMO_URL=https://... pnpm verify:public-demo-deployment` after any public static root is updated.
5. Record the restored version and manifest SHA in release notes before claiming a deployment is repaired.

## Current Limits

- No CDN cache invalidation or hosting-provider rollback has been executed.
- No public HTTPS deployment URL is attached for V4 examples.
- This artifact integrity plan supports local release-candidate review only; it does not satisfy the public deployment gate by itself.

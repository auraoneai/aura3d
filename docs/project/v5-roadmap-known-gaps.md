# V5 Known Gaps

> Historical note: This V5 document is retained as project history after the V9 parity reset. Current planning, claim boundaries, and code-backed parity status live in `docs/project/v9-roadmap-status.md`, `docs/project/v9-roadmap-parity-matrix.md`, and `docs/project/v9-roadmap-three-js-parity-plan.md`. Treat unchecked tasks or old claims here as historical unless they are restated in the V9 docs.


V5 starts from the completed V4 bounded SDK/product suite. The following remain gaps until proven by V5 release artifacts:

- Broad Three.js replacement for documented mainstream use cases.
- Full Three.js API parity.
- Full Three.js ecosystem replacement.
- High-end flagship visuals across product, automotive, architecture, materials, assets, animation, postprocess, VFX, large scenes, shader lab, and migrated Three.js scenes.
- Real HDR environment library with provenance.
- Real PBR material library with texture-set provenance.
- Broad loader ecosystem beyond the bounded V4 glTF proof.
- Controls equivalent to common Three.js example controls.
- Animation, skinning, morph, shader, postprocess, and VFX depth.
- Large-scene scale with instancing, culling, LOD, BVH, and profiling.
- Migration tooling and migrated Three.js example proof.
- External packed-package proof for every V5 public subpath and selected V5 templates.

These gaps must stay visible until the matching V5 reports pass.

## Working Rules

1. Do not claim Unity, Unreal, Babylon, broad Three.js, production, full glTF, full WebGPU, full PBR, full HDR, full shadows, or full postprocess parity.
2. Do not use stale V4 parity reports as engine-readiness completion evidence.
3. Every visual claim requires a screenshot generated by a browser test.
4. Every example must use the engine-readiness renderer or asset APIs.
5. Public examples stay limited to the V1 engine-readiness set until new examples pass equivalent screenshot gates.

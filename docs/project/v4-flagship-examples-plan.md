# V4 Flagship Examples Plan

> Historical note: This V4 document is retained as project history after the V9 parity reset. Current planning, claim boundaries, and code-backed parity status live in `docs/project/v9-roadmap-status.md`, `docs/project/v9-roadmap-parity-matrix.md`, and `docs/project/v9-roadmap-three-js-parity-plan.md`. Treat unchecked tasks or old claims here as historical unless they are restated in the V9 docs.


V4 should focus on a small number of strong examples instead of many weak ones.

## Portfolio Rule

- [x] The portfolio should feature only examples that pass V4 visual gates.
- [x] Old primitive examples may remain as basic tutorials, but they must not be presented as maximum capability.
- [x] Every portfolio card must link to a current screenshot generated by browser tests.
- [x] Every portfolio card must include known limits in machine-readable state, not marketing copy.

## Large World Streaming

Path: `examples/large-world-streaming`

- [x] Add browser screenshot and movement tests. Evidence: `tests/browser/large-world-streaming-v4.spec.ts` captures `tests/reports/v4-example-screenshots/large-world-streaming.png` and asserts camera-path movement.

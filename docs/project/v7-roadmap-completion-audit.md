# V7 Completion Audit

> Historical note: This V7 document is retained as project history after the V9 parity reset. Current planning, claim boundaries, and code-backed parity status live in `docs/project/v9-roadmap-status.md`, `docs/project/v9-roadmap-parity-matrix.md`, and `docs/project/v9-roadmap-three-js-parity-plan.md`. Treat unchecked tasks or old claims here as historical unless they are restated in the V9 docs.


This audit maps the current V7 objective to concrete artifacts. It is intentionally strict: a test, report, or screenshot only counts when it directly covers the requirement.

## Objective

G3D must move from low-quality renderer proof toward a real Three.js competitor. The current screenshot is not accepted as final quality. The required gaps are:

- true cubemap PMREM parity
- bounded same-scene contact/shadow delta parity plus a renderer-owned screen-space depth-contact postprocess, with full ray/general contact-shadow parity still open
- broad material-extension visual proof
- dedicated transmission, volume, anisotropy, iridescence, clearcoat, sheen, specular, IOR, dispersion, emissive-strength, and diffuse-transmission artifacts
- WebGPU as more than coverage/gap reporting
- broad Three.js API/ecosystem replacement proof
- official Three.js example-category parity for animation, skinning, additive blending, morphs, IK, decals, stereo/parallax, and physics

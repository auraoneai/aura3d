# Support Policy

Version: 1.0.0

## Current Support Level

Aura3D is an production TypeScript-first browser 3D engine and workflow SDK. Support for version `1.0.0` is limited to repository issues, reproducible test cases, and documentation updates.

## Where To Ask For Help

- Use the bug report template for reproducible defects.
- Use the feature request template for scoped enhancement proposals.
- Include the package version, commit SHA, browser, OS, reproduction command, and relevant report JSON path.

## Response Expectations

This rebuild does not yet have production service-level commitments. Maintainers should prioritize:

- release verification failures;
- security reports;
- public claim or documentation contradictions;
- regressions in examples, tests, and package exports.

## Known Limits

Known limitations belong in `docs/project/known-limits.md`, `docs/project/v9-roadmap-claim-boundary.md`, `docs/project/v9-roadmap-parity-matrix.md`, or the relevant historical roadmap document. Public support responses must not claim production readiness, full Three.js parity, stronger competitive positioning, full WebGPU support, Unity/Unreal replacement, or ecosystem completeness unless `docs/project/claim-guidelines.md` allows the exact wording.

Public support wording is also bound by `docs/project/v2-claim-registry.md`; use the registry when deciding whether a claim is allowed, blocked, or evidence-limited.

## Current V9 Support Boundary

Support questions about the current renderer should start from the V9 docs:

- `docs/project/v9-roadmap-status.md` for current code-backed evidence.
- `docs/project/v9-roadmap-parity-matrix.md` for matched, partial, unsupported, and exceeded coverage.
- `docs/project/v9-roadmap-three-js-parity-plan.md` for the implementation target.
- `docs/project/v9-roadmap-claim-boundary.md` for wording that must remain blocked.

V2 through V8 roadmap files are historical records after the V9 reset. They can explain why a subsystem exists, but they are not the current support contract unless V9 restates the same behavior.

## Current Status

V4 visual quality is tracked through browser screenshots, same-scene comparisons, gallery coverage, and visual QA reports.

The current gallery evidence is generated by:

```sh
pnpm v4:examples
```

## Required Gallery Categories

- product configurator,
- material studio,
- asset gallery,
- interior scene,
- character viewer,
- interactive showcase,
- installable template,
- gallery page,
- Three.js parity screenshot,
- parity diff image.

## Quality Gate

The current visual QA gate checks:

- screenshot files exist,
- PNG dimensions are readable,
- image files are not tiny placeholders,
- all required categories are present,
- same-scene Three.js parity and diff evidence are included,
- rejected V1 screenshots are not used as V4 gallery evidence.

## Boundary

The visual QA report validates gallery evidence for supported V4 workflows. It does not prove full renderer parity with Three.js, Unity, Unreal, or every glTF/PBR extension.
